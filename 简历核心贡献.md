# Trip Agent - AI 智能旅行规划助手

**项目地址**：https://github.com/liluo-llk/trip-agent

**技术栈**： Spring Boot + Spring AI + DeepSeek + PostgreSQL + pgvector + MCP + Caffeine + Reactor + SSE

**项目介绍**
一个基于 Plan-and-Execute + ReAct 混合架构的 AI 旅行规划助手，支持自动意图识别、多轮对话、工具调用、RAG 知识增强、流式响应。通过 MCP 协议解耦工具服务，采用双模型分层策略（规划用高温度创意模型、执行用低温度精确模型），配合向量检索注入本地人攻略，生成专业且个性化的旅行计划。

**核心贡献**

- **Plan-and-Execute + ReAct 混合架构**：设计分层 Agent 系统，主协调器编排规划与执行两个阶段，规划 Agent 使用完整 ReAct 循环调用工具收集信息并生成结构化旅行计划，执行 Agent 使用有限 ReAct（最多 3 轮）逐步执行单个计划步骤，实现全局规划与局部执行解耦，避免纯 ReAct 在复杂多步任务中的迷失问题。

- **双模型分层调度**：规划阶段采用高温度大模型保证创意多样性，执行阶段采用低温度轻量模型保证工具调用精确性，同一进程中通过依赖注入隔离两个模型实例，实现资源与策略解耦。

- **MCP 协议工具集成**：基于 Model Context Protocol 实现 Agent 与外部工具服务的进程级解耦，工具服务以子进程方式启动规避 HTTP 连接池泄漏问题；新增工具只需添加注解声明，零侵入扩展，已集成天气、景点、酒店、餐厅四类工具。

- **RAG 知识增强规划**：集成 pgvector 向量数据库 + 千问 Embedding 模型，启动时自动将 Markdown 攻略按语义分块向量化；规划时通过余弦相似度检索相关知识注入提示词，使输出融合本地人经验（避坑提示、私藏餐厅），而非仅依赖模型通用知识。

- **响应式流式架构**：基于 Reactor 的响应式流贯穿全链路，Agent 推理过程通过 SSE 实时推送五种事件（思考/规划/执行/工具调用/结果）；实现定时清理机制防止客户端异常断开导致的连接泄漏，保障长时间任务下的内存稳定。

- **阻塞调用非阻塞化**：LLM 调用和外部工具调用均为阻塞操作，通过弹性线程池隔离避免阻塞 Reactor 调度线程；递归式响应循环替代阻塞等待，配合最大迭代次数兜底防止死循环，单任务可支撑 10 轮以上推理。

- **工具调用容错体系**：工具调用层配置 3 次重试与指数退避策略；工具入参通过 ObjectMapper 安全序列化，杜绝字符串拼接导致的注入风险；工具服务端启用本地缓存 30 分钟，减少外部 API 调用频次，降低下游压力。

- **LLM 输出容错解析**：实现三层 JSON 提取策略（语义代码块 → 通用代码块 → 花括号匹配），应对模型输出格式不稳定问题；通过正则表达式统一修复枚举大小写与单复数变体，反序列化成功率从约 70% 提升至 99% 以上。

- **会话管理与记忆持久化**：基于并发安全容器管理多用户会话，支持多线程场景下的读写隔离；聊天记录持久化至 PostgreSQL，滑动窗口控制上下文长度避免 token 溢出；记录用户偏好与历史访问城市，为个性化规划提供数据支撑。

- **统一异常与数据质量治理**：设计分层错误码体系（AI 服务 / 外部 API / 业务逻辑 / 系统内部），对外返回用户友好提示、对内记录完整堆栈；数据结果标记来源等级（真实数据 / 降级数据 / 查询失败），下游可据此决定可信度与降级策略。
