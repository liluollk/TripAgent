# 熔断降级方案

## 背景

当前项目缺乏熔断降级机制：
- LLM 调用无重试
- Embedding 调用无重试、无降级
- 无熔断机制，服务不可用时会持续请求导致雪崩

---

## 方案设计

### 依赖

```xml
<!-- Resilience4j Circuit Breaker -->
<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot4</artifactId>
    <version>2.4.0</version>
</dependency>
```

> 注意：Spring Boot 4.x 需要使用 `resilience4j-spring-boot4`，不是 `resilience4j-spring-boot3`

---

### 配置

```yaml
# application.yml
resilience4j:
  circuitbreaker:
    instances:
      llm-cb:
        sliding-window-size: 10
        failure-rate-threshold: 50
        wait-duration-in-open-state: 30s
        permitted-number-of-calls-in-half-open-state: 3
        register-health-indicator: true
        automatic-transition-from-open-to-half-open-enabled: true
      embedding-cb:
        sliding-window-size: 10
        failure-rate-threshold: 50
        wait-duration-in-open-state: 60s
        permitted-number-of-calls-in-half-open-state: 2
      mcp-cb:
        sliding-window-size: 5
        failure-rate-threshold: 60
        wait-duration-in-open-state: 30s
        permitted-number-of-calls-in-half-open-state: 2
  retry:
    instances:
      llm-retry:
        max-attempts: 3
        wait-duration: 1s
        enable-exponential-backoff: true
        exponential-backoff-multiplier: 2
      embedding-retry:
        max-attempts: 3
        wait-duration: 500ms
        enable-exponential-backoff: true
        exponential-backoff-multiplier: 2
```

### 代码修改

#### 1. CircuitBreakerConfig.java (新建)

```java
package com.tripagent.config;

import io.github.resilience4j.circuitbreaker.CircuitBreaker;
import io.github.resilience4j.circuitbreaker.CircuitBreakerRegistry;
import io.github.resilience4j.retry.RetryRegistry;
import lombok.extern.slf4j.Slf4j;
import org.springframework.context.annotation.Configuration;
import jakarta.annotation.PostConstruct;

@Slf4j
@Configuration
public class CircuitBreakerConfig {

    private final CircuitBreakerRegistry circuitBreakerRegistry;
    private final RetryRegistry retryRegistry;

    public CircuitBreakerConfig(
            CircuitBreakerRegistry circuitBreakerRegistry,
            RetryRegistry retryRegistry) {
        this.circuitBreakerRegistry = circuitBreakerRegistry;
        this.retryRegistry = retryRegistry;
    }

    @PostConstruct
    public void init() {
        log.info("=== 熔断降级配置初始化 ===");
        circuitBreakerRegistry.getAllCircuitBreakers().forEach(cb -> {
            log.info("熔断器 [{}]: 状态={}, 失败率阈值={}%",
                    cb.getName(), cb.getState(),
                    cb.getCircuitBreakerConfig().getFailureRateThreshold());
        });
        retryRegistry.getAllRetries().forEach(retry -> {
            log.info("重试器 [{}]: 最大重试次数={}",
                    retry.getName(), retry.getRetryConfig().getMaxAttempts());
        });
        log.info("=== 熔断降级配置完成 ===");
    }
}
```

    // 原有逻辑

// 降级方法 - 返回已收集的信息
private String combineResultsFallback(String userRequest, Map<Integer, String> stepResults, Throwable t) {
    log.warn("LLM 组合结果降级: {}", t.getMessage());
    StringBuilder fallback = new StringBuilder();
    fallback.append("## 旅行信息收集结果\n\n");
    fallback.append("⚠️ **注意**: AI 生成服务暂时不可用，以下是已收集的原始信息。\n\n");
    for (Map.Entry<Integer, String> entry : stepResults.entrySet()) {
        fallback.append("### 查询结果 ").append(entry.getKey()).append("\n");
        fallback.append(entry.getValue()).append("\n\n");
    }
    fallback.append("---\n");
    fallback.append("*AI 行程生成服务暂时不可用，请稍后重试以获取完整行程规划。*");
    return fallback.toString();
}

## 熔断器状态说明

```
CLOSED (正常) → 请求正常通过
    ↓ 失败率 > 阈值
OPEN (熔断) → 请求被拒绝，走降级逻辑
    ↓ 等待 30s/60s
HALF_OPEN (半开) → 允许少量请求尝试
    ↓ 成功
CLOSED (恢复正常)
    ↓ 失败
OPEN (继续熔断)
```

---

## 降级策略

| 组件 | 降级行为 |
|------|----------|
| LLM (PlanningAgent) | 返回默认计划（天气+景点+酒店+餐厅） |
| LLM (ExecutionAgent) | 返回已收集的原始信息 + 错误提示 |
| Embedding | 返回零向量 → RagService 跳过 RAG |
| MCP 工具 | 返回友好错误提示 |

---

## 测试方法

1. 启动服务，正常请求 → 正常响应
2. 停止 MCP 工具服务器 → 返回工具降级结果
3. 配置错误的 LLM API Key → 返回 LLM 降级结果
4. 配置错误的 Embedding API Key → 跳过 RAG 直接回答

---

## 注意事项

1. Spring Boot 4.x 必须使用 `resilience4j-spring-boot4`，不是 `resilience4j-spring-boot3`
2. 不需要显式添加 `spring-boot-starter-aop`，Resilience4j 会自动引入
3. 降级方法的参数必须与原方法一致，外加 `Throwable` 参数
4. `@Retry` 注解必须在 `@CircuitBreaker` 之前（先重试，再熔断）
